## 4. 프로토콜 특성 요약

### 4.1 OLSR (Optimized Link State Routing)

| 항목 | 특성 |
|------|------|
| 방식 | 프로액티브 (Proactive) |
| 경로 구축 | 사전 구축 (HELLO + TC) |
| 장점 | 즉시 전송 가능, 전체 토폴로지 파악 |
| 단점 | 높은 오버헤드, 변화 반영 지연 |
| 적합 환경 | 대역폭 여유, 낮은 이동성 |

### 4.2 AODV (Ad-hoc On-demand Distance Vector)

| 항목 | 특성 |
|------|------|
| 방식 | 리액티브 (Reactive) + 캐시 |
| 경로 구축 | 요청 시 탐색 (RREQ/RREP) |
| 장점 | 최신 경로 사용, 링크 검증 |
| 단점 | 탐색 지연, 높은 에너지 소비 |
| 적합 환경 | 간헐적 트래픽, GPS 없는 환경 |

### 4.3 GPSR (Greedy Perimeter Stateless Routing)

| 항목 | 특성 |
|------|------|
| 방식 | 지리적 (Geographic) |
| 경로 구축 | 없음 (위치 기반 즉시 결정) |
| 장점 | 최저 지연/오버헤드/에너지 |
| 단점 | GPS 필요, void 문제 |
| 적합 환경 | 위치 정보 가용, 에너지 제한 |
