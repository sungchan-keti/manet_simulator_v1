
| 프로토콜 | PDR | 전송 | 수신 | 드랍 |
|---------|-----|------|------|------|

| OLSR | 86.75% | 3,585 | 3,110 | 475 |

**분석**:

**AODV (100.00%)** - 최고 PDR
- 리액티브 방식으로 경로 요청 시 최신 경로 탐색
- 링크 유효성 검사로 끊긴 링크 즉시 감지 및 재탐색
- 캐시 최적화로 안정적인 경로 유지

**GPSR (97.99%)** - 2위
- 위치 기반 greedy 포워딩으로 대부분 성공
- 72개 드랍 = void 상황 (모든 이웃이 목적지에서 더 멂)
- Network Connected: True (안정적 토폴로지)

**OLSR (86.75%)** - 3위
- Network Connected: False (시뮬레이션 종료 시 네트워크 분리)
- 475개 드랍 = 분리된 노드로의 전송 실패 + stale 경로
- TC 간격(2초) 동안 토폴로지 변화 반영 불가

## 4. 프로토콜 특성 요약

### 4.1 OLSR (Optimized Link State Routing)

| 항목 | 특성 |
|------|------|
| 방식 | 프로액티브 (Proactive) |
| 경로 구축 | 사전 구축 (HELLO + TC) |
| 장점 | 최저 지연, 전체 토폴로지 파악 |
| 단점 | 높은 오버헤드, 네트워크 분리 시 PDR 저하 |
| 적합 환경 | 대역폭 여유, 안정적 토폴로지 |

**적용된 최적화**:
- 링크 유효성 검사: 다음 홉이 실제 이웃인지 확인
- 효과: stale 경로 사용 방지

### 4.2 AODV (Ad-hoc On-demand Distance Vector)

| 항목 | 특성 |
|------|------|
| 방식 | 리액티브 (Reactive) + 캐시 |
| 경로 구축 | 요청 시 탐색 (RREQ/RREP) |
| 장점 | 100% PDR, 최신 경로 보장, 위치 정보 불필요 |
| 단점 | 경로 탐색 지연, 변동이 큰 지연 |
| 적합 환경 | GPS 없는 환경, 높은 신뢰성 요구 |

**적용된 최적화**:
- 경로 캐시: ROUTE_TIMEOUT 30초, 사용 시 연장
- 링크 유효성 검사: 다음 홉이 이웃인지 확인
- 효과: 경로 탐색 지연 대폭 감소

### 4.3 GPSR (Greedy Perimeter Stateless Routing)

| 항목 | 특성 |
|------|------|
| 방식 | 지리적 (Geographic) |
| 경로 구축 | 없음 (위치 기반 즉시 결정) |
| 장점 | 최저 오버헤드/에너지, 안정적 지연 |
| 단점 | GPS 필요, void 문제로 일부 드랍 |
| 적합 환경 | 위치 정보 가용, 에너지 제한 |
